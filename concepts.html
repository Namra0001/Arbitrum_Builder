Web3 Concepts

Side-by-side comparisons to help you understand the building blocks of decentralized technology.

Web2 vs Web3

Web2

The Centralized Web

  • Platforms like Google and Facebook own your data
  • Companies control servers and can censor content
  • You need permission to build on their platforms
  • Revenue flows to platform owners, not creators
  • Single points of failure — if a server goes down, the app dies
Web3

The Decentralized Web

  • You own your data through wallets and on-chain identity
  • No single entity controls the network — it's peer-to-peer
  • Anyone can deploy smart contracts without asking permission
  • Token incentives reward users and developers directly
  • Resilient by design — no central server to take down

Ethereum vs Bitcoin

Bitcoin

Digital Gold

  • Created in 2009 as peer-to-peer digital cash
  • Primary purpose: store of value and payments
  • Scripting language is intentionally limited
  • ~7 transactions per second on mainnet
  • Proof of Work consensus (energy-intensive mining)
Ethereum

Programmable Blockchain

  • Launched in 2015 as a "world computer"
  • Runs smart contracts — self-executing code on-chain
  • Turing-complete — can build any application logic
  • ~15 TPS on L1, thousands on Layer 2 like Arbitrum
  • Now uses Proof of Stake (energy-efficient validation)

Public Key vs Private Key

Private Key

Your Secret Password

  • A 256-bit random number — never share it with anyone
  • Proves you own the assets in your wallet
  • Used to sign transactions cryptographically
  • Lost private key = lost funds forever (no reset button)
  • Usually stored as a 12–24 word seed phrase
Public Key

Your Wallet Address

  • Derived from your private key using elliptic curve math
  • Safe to share — it's your receiving address (0x...)
  • Anyone can send crypto to your public key
  • Cannot be reverse-engineered to reveal the private key
  • One private key can generate many public addresses

Blockchain vs Traditional Databases

Traditional DB

Centralized Storage

  • One organization controls read, write, and delete access
  • Data can be modified or deleted by admins at any time
  • Fast — optimized for millions of queries per second
  • Trust the company running the database
  • Examples: MySQL, PostgreSQL, MongoDB
Blockchain

Shared, Immutable Ledger

  • Thousands of nodes maintain identical copies of data
  • Once written, records cannot be altered — only appended
  • Slower by design — consensus takes time
  • Trust the math and code, not a single authority
  • Examples: Ethereum, Bitcoin, Arbitrum
